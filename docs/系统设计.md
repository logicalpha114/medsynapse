```
                     │ REST API (JSON)

│              后端 FastAPI (Python)                    │

        │                    │

│ parsed/       │  │ data/vector_db/     │
│ summaries/    │  └────────────────────┘
└──────────────┘
```

### 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | 异步支持好、自动生成API文档、Python生态完善 |
| PDF解析 | PyMuPDF (fitz) | 速度快、支持逐页流式、中文兼容好 |
| 大模型 | DeepSeek Chat API | OpenAI兼容接口、中文能力强、价格低 |
| 向量嵌入 | BGE-small-zh-v1.5 | 中文专用、本地免费、模型小(100MB) |
| 向量库 | ChromaDB | 内置元数据过滤、持久化、轻量级 |
| 前端 | HTML+JS+ECharts | 零构建、CDN加速、ECharts图能力强大 |
| 部署 | 魔搭创空间 / uvicorn | 免费CPU、支持Python |

## 2. 数据流

### 教材解析流程

```
PDF文件 → PyMuPDF逐页读取 → 过滤页眉页脚/图片
       → 正则匹配"第X章"识别章节
       → 输出: { textbook_id, title, chapters[{ chapter_id, title, content, page }] }
```

### 知识图谱构建流程

```
结构化JSON → 逐章提交LLM提取知识点(8-15个/章)
           → 逐章提交LLM识别关系(4种关系类型)
           → 生成章级摘要MD + 书级摘要MD
           → 输出: { nodes[], relations[] }
```

### 跨教材整合流程

```
所有KG节点 → BGE-small-zh向量化 → 余弦相似度矩阵
           → 筛选≥0.75候选对 → 高相似度(≥0.90)自动合并
           → 其余提交LLM精判(merge/keep/remove)
           → 压缩控制(目标≤30%) → 输出整合结果
```

### RAG检索流程

```
用户问题 → BGE向量化 → ChromaDB检索(先摘要层后原文层)
        → 合并去重 top-5 → 拼接上下文
        → 注入RAG Prompt → DeepSeek生成带引用回答
```

## 3. API 接口一览

| 端点 | 方法 | 请求 | 响应 |
|------|------|------|------|
| `/api/textbooks` | GET | - | `{ textbooks[], count }` |
| `/api/textbooks/upload` | POST | FormData(file) | `{ textbook_id, filename, status }` |
| `/api/textbooks/{id}/parse` | POST | - | `{ textbook_id, chapters[], total_pages }` |
| `/api/textbooks/{id}/kg` | POST | - | `{ node_count, relation_count }` |
| `/api/kg/{id}` | GET | - | `{ nodes[], relations[] }` |
| `/api/integrate` | POST | - | `{ decisions[], compression_ratio }` |
| `/api/rag/index` | POST | - | `{ total_chunks, textbooks_indexed }` |
| `/api/rag/query` | POST | `{ question }` | `{ answer, citations[], source_chunks[] }` |
| `/api/rag/status` | GET | - | `{ indexed, textbooks_count }` |
| `/api/dialogue` | POST | `{ message }` | `{ reply, action_taken }` |
| `/api/report` | GET | - | `{ summary, decisions_summary, cases[] }` |
