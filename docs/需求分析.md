# 需求分析

## 1. 问题分解

### 核心问题

7 本医学教材（约2500页，>100万字）存在大量内容重复，需要 AI 帮助教师将其整合压缩到不超过原始体量 30% 的精华版本，且教学效果不打折。

### 子问题拆解

#### 1.1 知识点粒度定义

- **知识点**：医学教材中一个可独立教学的核心概念、原理、机制或方法
- **粒度标准**：能在2-3句话内完整定义，是教学的原子单元
- **示例**："动作电位"是一个知识点，"静息电位"是另一个，它们之间存在前置依赖关系
- **范围**：每章预期提取8-15个核心知识点，全7本书预期1500-2500个知识点

#### 1.2 重复判定标准

采用二级判定机制：

- **一级（语义相似度）**：BGE-small-zh-v1.5 计算向量余弦相似度，≥0.75 进入候选
- **二级（LLM精判）**：DeepSeek Chat 判断两个知识点是否等价，输出 merge/keep + 理由 + 置信度
- **高置信度自动决策**：相似度 ≥0.90 直接合并，跳过 LLM 调用以节省时间和 Token

#### 1.3 教学连贯性保障

- 保留知识点之间的语义关系（前置依赖、并列、包含、应用）
- 合并时选择描述最完整的版本作为主版本
- 删除决策仅针对"完全冗余"的节点（其他教材已有更详细描述）
- 整合后保留关系链，确保学习路径不断裂

#### 1.4 压缩比计算方式

- 压缩比 = 整合后知识点总字数 / 原始知识点总字数
- 压缩方式：合并重复节点 + 删除冗余节点 + 定义文字优化
- 目标：≤30%

#### 1.5 RAG 分块策略选择依据

- **块大小 600 字**：中文教材自然段落500-800字，600字覆盖一个完整概念解释
- **重叠 100 字**：约2-3句话，防止知识点在块边界被截断
- **按段落切分**：不在句子中间截断，保证语义完整
- **两层摘要**：先搜摘要定位教材/章节，再搜原文 chunk 精确定位

## 2. 约束条件

| 约束 | 说明 |
|------|------|
| 时间 | 5小时开发时间 |
| 教材 | 7本医学PDF，约2500页，>100万字 |
| 格式 | 必须支持PDF/MD/TXT |
| 部署 | 必须公网可访问 |
| 压缩 | 整合后≤原始30% |
| 引用 | RAG回答必须带来源引用 |

## 3. 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| PDF章节识别不准 | 知识图谱质量下降 | 正则+字体信息双重判断；对话接口允许修正 |
| LLM API 限流/超时 | 进度卡死 | 每个章节独立处理，失败只影响单章 |
| 压缩比达不到30% | 不符合赛题要求 | 追加定义截断策略；优先合并重复内容 |
| ChromaDB 内存占用 | 服务器OOM | BGE-small模型仅100MB；分批向量化 |
